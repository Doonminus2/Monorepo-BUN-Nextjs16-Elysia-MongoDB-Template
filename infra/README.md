# Infrastructure (โครงสร้างระบบ)

โฟลเดอร์นี้ใช้สำหรับเก็บการตั้งค่าที่เกี่ยวข้องกับโครงสร้างระบบ (Infrastructure)
ซึ่งแยกออกจากโค้ดของแอปพลิเคชันโดยตั้งใจ

## วัตถุประสงค์

- แยก concerns ระหว่าง application กับ infrastructure
- ทำให้สามารถเปลี่ยนหรือแทนที่บริการภายนอกได้โดยไม่กระทบโค้ดหลัก
- รองรับการพัฒนาแบบ local development และขยายไปสู่ production ในอนาคต

## โครงสร้าง

- `docker/` – การตั้งค่า Docker สำหรับบริการที่ใช้ใน local development

## แนวทางการใช้งาน

- โค้ดใน `apps/` ไม่ควรพึ่งพารายละเอียดของ infra โดยตรง
- ตัวแปร environment ให้จัดการในระดับของแต่ละแอป (เช่น `apps/api/.env`)
- infra ควรถือว่าเป็นสิ่งที่สามารถถูกแทนที่ได้ (replaceable)
